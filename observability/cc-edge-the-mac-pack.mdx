> The Mac side of the OS telemetry stack. The macOS equivalent of `Splunk_TA_nix` — delivered through Cribl Edge instead of scripted inputs on a forwarder.

<RepoMeta language="Shell" status="active" lastActive="this week" repoUrl="https://github.com/JacobPEvans/cc-edge-the-mac-pack-io" />

`cc-edge-the-mac-pack-io` is a Cribl Edge pack that collects macOS host telemetry — unified logs, system metrics, thermal pressure, per-process energy, battery health, and snapshot files. As of v0.3.0 it uses Cribl 4.18's native macOS Sources (`apple_unified_logs`, `system_metrics`) for the broad capture and falls back to exec sources only for data the native ones don't yet expose. It supersedes two predecessor repos (`cc-edge-macos-system` and `cc-edge-macos-power`, both archived).

## Design rule

**Edge captures broadly, Stream filters.** This pack pulls every Apple subsystem's unified log and every host metric the native Sources expose. Per-use-case predicates — WindowServer ping detection, Jetsam triage, security event extraction — belong in Stream pipelines downstream, not in this pack.

## Targeted Edge install

The pack targets a **native macOS Cribl Edge install** (Nix-managed). Exec inputs that invoke macOS-only binaries (`powermetrics`, `pmset`, `ioreg`, `memory_pressure`) require host access and will not work from a Linux container or the OrbStack-deployed cluster. The file input (`mac-perf-snapshots`) also reads from a Mac filesystem path.

## Data sources

| Input | Type | What it captures | Index |
| --- | --- | --- | --- |
| `in_apple_unified_logs` | Native 4.18 Source | All Apple subsystems (kernel, WindowServer, security, network, power, Spotlight, Time Machine, TCC, sandbox, launchservices, xpc, …) via `subsystem BEGINSWITH "com.apple"` | `os` |
| `in_system_metrics` | Native 4.18 Source | Host CPU, memory, disk, network, process (60s poll) | `mac_perf` |
| `macos-thermal` | Exec source | `pmset -g therm` — thermal/perf warning levels, CPU power status (60s) | `os` |
| `macos-power-metrics` | Exec source (root) | `powermetrics` — per-process energy, CPU/GPU/ANE power, thermal pressure (300s) | `mac_perf` |
| `macos-power-battery` | Exec source | `pmset -g batt` + `ioreg -r -c AppleSmartBattery` — charge, cycles, capacity, temperature (60s) | `os` |
| `mac-perf-snapshots` | File source | NDJSON snapshot files emitted by an external collector | `mac_perf` |

## Anomaly detection

Edge keeps anomaly detection only for the signals that exec sources produce — anomaly detection for memory pressure, WindowServer ping timeouts, and Jetsam events moves to Stream-side pipelines downstream of this pack.

| Condition | `anomaly_reason` | Detected at |
| --- | --- | --- |
| `battery_health_percent < 80` | `battery_health_degraded` | Edge (this pack) |
| Thermal pressure not Nominal | `thermal_pressure_elevated` | Edge (this pack) |
| Memory pressure critical | (Stream-side, future) | Stream |
| WindowServer ping timeout | (Stream-side, future) | Stream |
| Jetsam event | (Stream-side, future) | Stream |

Alert on what Edge does flag:

```spl
(index=os OR index=mac_perf) anomaly=true
| stats count by host, anomaly_reason, _time
```

## How it fits

| Upstream | This | Downstream |
| --- | --- | --- |
| macOS unified logs, system metrics, `pmset`, `powermetrics`, `ioreg`, NDJSON snapshot files | This pack — broad capture only | Cribl Stream pipelines (per-use-case filtering); Splunk (`index=os`, `index=mac_perf`) |

## Customizing without forking

Local overrides land in `/opt/cribl/local/cc-edge-the-mac-pack-io/inputs.yml` — Cribl Edge merges them on top of the pack defaults:

```yaml
inputs:
  in_apple_unified_logs:
    predicate: 'TRUEPREDICATE'   # capture 3rd-party app logs too
  in_system_metrics:
    pollingInterval: 30          # increase metric frequency
  macos-power-metrics:
    disabled: true               # disable if root unavailable
  mac-perf-snapshots:
    disabled: true               # disable if no external collector is running
```

## Installation

```bash
cp cc-edge-the-mac-pack-io.crbl /opt/cribl/state/packs/

curl -X POST http://localhost:9000/api/v1/packs \
  -H "Content-Type: application/json" \
  -d '{"source":"cc-edge-the-mac-pack-io.crbl"}'

curl -X POST http://localhost:9000/api/v1/version/commit
curl -X POST http://localhost:9000/api/v1/system/settings/restart
```

## v0.3.0 — the architecture shift

Six v0.2 exec inputs retired in favor of the native 4.18 Sources: `macos-memory-pressure`, `macos-vm-stat`, `macos-disk-io`, `macos-process-top`, `macos-windowserver-health`, `macos-jetsam-events`. `minLogStreamVersion` bumped to 4.18.0. Three exec inputs retained where no native equivalent exists: `macos-thermal`, `macos-power-metrics`, `macos-power-battery`. **Breaking** for any v0.2 dashboard filtering `sourcetype=macos:system:memory|windowserver|jetsam|process|diskio|vmstat` — that data is now under `macos:unified_log` and `macos:system:metrics`.
