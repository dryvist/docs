> The dashboards layer for AI coding tool telemetry. Six dashboards, one base search pattern, every metric you want from a Claude or Gemini session.

<RepoMeta language="JSON" status="active" lastActive="this week" repoUrl="https://github.com/JacobPEvans/VisiCore_App_for_AI_Observability" />

`VisiCore_App_for_AI_Observability` is a Splunk App that ships six Dashboard Studio v2 dashboards covering every angle of AI-coding-tool usage: a roll-up overview, token consumption by type, dollar cost by model/session/project, tool-call patterns, per-session detail, and prompt-cache performance. It pairs with the [VisiCore TA](/observability/visicore-ta), which provides the field extractions, macros, and lookups every dashboard depends on.

## What it does

- Ships six Dashboard Studio v2 dashboards (JSON CDATA) targeting `index=claude` and `index=gemini`
- Uses the base-search + extend pattern so a single dashboard run scans the index once
- Reads all knowledge objects (props, transforms, macros, lookups) from the companion TA â€” no duplication
- All config lives in `default/` only (no `local/`) for Splunk Cloud compatibility
- Repo root IS the Splunk package root â€” `default/`, `metadata/`, `static/` at the top level

## The six dashboards

| Dashboard | What it answers |
| --- | --- |
| Overview | KPIs, token trends, model distribution, tool usage, session activity at a glance |
| Token Usage | Drill-down across input, output, cache-read, cache-creation tokens |
| Cost Analysis | Dollar cost by model, session, and project (lookup-driven pricing) |
| Tool Activity | Tool-call patterns and file-operation breakdown |
| Sessions | Per-session analysis â€” duration, project attribution, model used |
| Cache Performance | Prompt-cache efficiency and estimated token savings |

## Token model

Token names follow [`ccusage`](https://github.com/ryoppippi/ccusage) â€” `input_tokens`, `output_tokens`, `cache_read_input_tokens`, `cache_creation_input_tokens`. Dashboards in this App assume the TA has already extracted those fields and dedup'd on `messageId:requestId`.

## Getting started

<Steps>
  <Step title="Install the TA first">
    `splunk install app VisiCore_TA_AI_Observability-*.tar.gz`. The App will not render without it.
  </Step>
  <Step title="Install the App">
    `splunk install app VisiCore_App_for_AI_Observability-*.tar.gz && splunk restart`.
  </Step>
  <Step title="Ensure indexes exist">
    `claude` and `gemini` must be defined on the indexer tier. `ansible-splunk` provisions both.
  </Step>
  <Step title="Configure data inputs">
    Cribl Edge packs ship the data â€” see the [observability overview](/observability/overview) for the per-tool pack list.
  </Step>
</Steps>

## Packaging

`./scripts/package.sh` builds a versioned tarball in `build/`. CI validates the manifest with `jq . app.manifest`; `ansible-splunk` consumes the tarball and deploys it onto the Splunk install.
