> Active IaC workspaces do not use aws-vault.

`aws-vault` can store long-lived AWS access keys in an operating-system
keychain and mint bounded subprocess sessions. It remains useful for isolated
manual AWS administration where no platform identity exists.

It is not part of the dryvist OpenTofu path. Terrakube runs plans and applies;
OpenBao mints short-lived STS credentials for each workspace through native
dynamic credentials. Do not add aws-vault profiles, wrapper commands, or
long-lived operator keys to an active IaC repo.

Use [Consuming AWS workspace setup](/infrastructure/tofu-aws/consuming-repo)
for the current pattern.
