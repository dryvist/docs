## How they depend on each other

**Security gates everything.** Credentials reach CI through `secrets-sync`. Branch protection and rulesets are enforced declaratively by `.github-tofu` across both orgs. Every other layer assumes this layer is in place.

**Infrastructure provisions Configuration.** `terraform-proxmox` builds the VMs and LXCs on the Proxmox cluster. Ansible roles then install Cribl, HAProxy, and the application layer on top.

**Configuration deploys Observability.** Splunk Enterprise via `ansible-splunk`. Cribl Edge and Stream via `ansible-proxmox-apps`. The Cribl packs and Splunk apps live on this infrastructure.

**Nix tools AI Development.** Every AI coding tool — Claude Code, Gemini CLI, Copilot, MLX, MCP servers — is packaged in `nix-ai`. The rest of the Nix layer (`nix-darwin`, `nix-home`, `nix-devenv`) makes the entire dev environment rebuildable from a single `nix build`.

**AI Development emits to Observability.** Every AI session ships OTEL telemetry. Cribl collects, Splunk indexes, the AI-observability apps make it readable. End-to-end traceability for every model call.
