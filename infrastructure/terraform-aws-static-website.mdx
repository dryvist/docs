> A static-site module that defaults to encrypted, versioned, logged. No knobs to forget.

<RepoMeta language="HCL" status="active" lastActive="this week" repoUrl="https://github.com/JacobPEvans/terraform-aws-static-website" />

`terraform-aws-static-website` is a reusable module for AWS-hosted static sites. CloudFront in front, S3 behind, ACM for TLS, Route53 for DNS. Every bucket is encrypted, every bucket is versioned, every request is logged. The defaults are the audit-passable shape; SPA mode and Lambda@Edge are opt-in for the cases that need them.

## What it does

- Provisions CloudFront distributions with TLS 1.2+ and ACM wildcard certificates
- Creates the website bucket and a separate access-log bucket — both AES256-encrypted, versioning enabled
- Wires up Route53 A records to CloudFront; supports a redirect-domain (e.g. `www → root`) plus extra aliases
- Optional SPA mode flips 404s to `index.html` with 200 status for client-side routing
- Optional Lambda@Edge attachment for custom security headers (HSTS, CSP) or per-request behavior

## How it fits

| Provides | Consumers |
| --- | --- |
| A drop-in module callable from any Terraform/OpenTofu config | Project sites, landing pages, marketing pages — anything that's static HTML on S3 + CloudFront |

A module, not a deployment. Consume it from a calling Terraform repo that owns the per-site state.

## Important: us-east-1 provider

The module requires an aliased `aws.us-east-1` provider because ACM certificates for CloudFront must live in `us-east-1` regardless of where the rest of your stack runs. This is an AWS constraint, not a module choice.

## Getting started

<Steps>
  <Step title="Reference the module from a calling Terraform config">
    `source = "github.com/jacobpevans/terraform-aws-static-website"` — pin to a release tag in production.
  </Step>
  <Step title="Supply domains and the us-east-1 provider">
    `website-domain-main`, `domains-zone-root`, and optionally `website-domain-redirect`. Pass `providers = { aws.us-east-1 = aws.us-east-1 }`.
  </Step>
  <Step title="Apply">
    Standard `terraform apply` flow from the calling repo. The plan shows every bucket, distribution, certificate, and DNS record.
  </Step>
  <Step title="Upload content">
    Sync your built site to the website bucket the module outputs; CloudFront serves it within an invalidation cycle.
  </Step>
</Steps>

## Testing

The module ships with [LocalStack](https://localstack.cloud/) + [Terratest](https://terratest.gruntwork.io/) integration. `make test-unit` runs the fast checks; `make test-local` spins up LocalStack and runs the full integration suite. CI mirrors both.
